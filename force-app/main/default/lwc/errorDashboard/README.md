# System Diagnostics Dashboard (Agentforce Integrated)

An advanced Salesforce Lightning Web Component (LWC) and AI Agent application designed to monitor, filter, and autonomously diagnose system errors. 

This project bridges the gap between traditional custom UI development and the next generation of Salesforce AI by embedding an Agentforce "Diagnoser" subagent directly into the developer workflow.

## Key Features

* **Real-Time Log Filtering:** A custom LWC that displays active `Error_Log__c` records with instant, client-side search filtering across error names and stack traces.

![High Severity - LWC](images/LWC%20High%20Severity.png)

* **Dynamic SLDS UI:** Utilizes custom CSS and SLDS utility classes to render high-contrast severity badges (High/Medium/Low) and monospaced code blocks for easy scan-reading.

* **Agentforce Integration:** Features a custom Prompt Template grounded in the Error Log object, packaged as an Agent Action.

![Agentforce - Action Configuration](images/Agentforce%20-%20Action.png)

* **Autonomous Diagnostics:** Users can open the Einstein/Agentforce panel on any error record and ask *"How do I fix this?"* to receive a step-by-step root cause analysis and remediation plan generated by the LLM.

![Agent System Diagnoser](images/System%20Diagnoser%20Agent.png)

![Agent Response](images/Agent%20Response.png)

 **Email Alert:** Sends an automated email to the designated System Administrator group, when Severity is 'High'.

 ![Email Alert](images/Send%20Email.png)

 ![Email](images/Test%20email%20alert%20.png)


## Tech Stack

* **Frontend:** Lightning Web Components (LWC), JavaScript, HTML, custom CSS
* **Backend:** Apex (`ErrorLogController`), SOQL
* **AI & Automation:** Agentforce Builder, Prompt Builder, Copilot Actions
* **Data Model:** Custom Object (`Error_Log__c`)

## Setup & Installation

1. **Deploy Metadata:** Deploy the `errorDashboard` LWC and `ErrorLogController` Apex class to your org.
2. **Assign Permissions:** Ensure your user profile has the `Access Agentforce Default Agent` and `Agentforce Default Admin` permission sets.
3. **Activate the Agent:** * Navigate to **Agentforce Builder** in Setup.
    * Verify the custom `Diagnoser` subagent is active.
    * Ensure the **Diagnose Error Log** action is attached to the subagent with the instruction: *"Always use the Diagnose Error Log action to retrieve information and explain the root cause of the error to the user."*
4. **Page Layout:** Add the `errorDashboard` component to your preferred Lightning App Home Page or specific Record Page using the Lightning App Builder.

## Usage

1. Navigate to the **System Diagnostics Workspace** tab.
2. Use the search bar to filter logs by specific keywords (e.g., "CPU limit" or "504").
3. Click into a specific Error Log record.
4. Open the Agentforce Chat icon in the top global header.
5. Ask the agent: *"Diagnose this error"* and watch the AI break down the fix!
